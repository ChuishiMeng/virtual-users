# P1: 问题重新定位
> 基于KDD方法论：从失效场景到能力需求

---

## 一、现有方法局限性分析

### 1.1 四类方法的失效场景与根本原因

| 方法类别 | 核心假设 | 失效场景 | 根本原因 |
|---------|---------|---------|---------|
| **传统调研** | 人类受访者可大规模、长时间获取 | 预算有限、时间紧迫、敏感话题难以触达 | **依赖真实人类参与**：成本高、周期长、覆盖受限 |
| **直接LLM** | 模型具备通用社会认知能力 | 人口学对齐任务（如按年龄/收入分层模拟） | **缺乏人群分布意识**：模型趋向"平均人"，无法体现群体差异 |
| **简单Persona** | 文字描述足以定义一个完整角色 | 复杂态度问题、需要领域知识的议题 | **缺乏知识和记忆**：Persona是空壳，没有背景知识支撑 |
| **现有虚拟用户系统** | 单次推理即可生成可靠响应 | 需要跨问题态度一致性的场景 | **缺乏一致性保证**：问答之间可能矛盾，缺乏内部校验 |

### 1.2 失效场景详细分析

#### 场景1: 人口学分布对齐
```
问题: 模拟某地区选民的政治态度分布

直接LLM方法:
- 输入: "请模拟一个35岁、收入5万的女性选民"
- 输出: 倾向于给出"理性、中立"的回答
- 失效: 无法体现该人口学群体的真实态度分布（可能更偏向某政党）

根本原因: 
LLM在训练时学到的是"如何回答问题"，而非"某类人会如何回答"
→ 缺乏"Persona → 行为分布"的映射能力
```

#### 场景2: 领域知识落地
```
问题: 模拟糖尿病患者对某新药的看法

简单Persona方法:
- Persona: "55岁男性，患有2型糖尿病5年"
- 输出: 泛泛而谈，缺乏具体细节
- 失效: 无法引用真实的药物信息、副作用、治疗经验

根本原因:
Persona描述仅定义了"身份标签"，未注入"领域知识"
→ 模型无法区分"我是糖尿病患者"和"我了解糖尿病治疗"
```

#### 场景3: 跨问题态度一致性
```
问题: 消费者调研中的关联问题

Q1: 您购买手机时最看重什么？
A1: "性价比，不追求最新款"

Q2: 您最近购买了哪款手机？
A2: "iPhone 15 Pro，刚发布就买了"

失效: A1和A2明显矛盾，但系统无法检测

根本原因:
单次独立推理 → 每个问题独立生成 → 无跨问题约束机制
```

## 二、新能力需求

基于上述失效分析，可靠的虚拟用户系统需要四大能力：

### 能力1: 人群感知能力（统计落地）
```
定义: 模型需理解不同人口学群体的态度分布差异

具体要求:
- 给定人口学特征，能预测态度倾向
- 理解"年龄×收入×教育"的交互效应
- 从真实调研数据中学习群体分布

技术路径: Persona残差表示
  p_i^res = p_i - E[p | demographics]
  → 消除全局偏差，保留群体特性
```

### 能力2: 知识落地能力（领域落地）
```
定义: Persona需要领域知识支撑，而非空洞身份标签

具体要求:
- 根据Persona自动检索相关领域知识
- 支持多领域（政治、医疗、消费、教育）
- 知识与Persona身份匹配（如患者应有医疗知识）

技术路径: RAG + 领域知识库
  Knowledge(Persona, Question) → 注入到生成过程
```

### 能力3: 一致性保证能力（跨问题约束）
```
定义: 虚拟用户在不同问题间保持态度一致

具体要求:
- 检测跨问题矛盾（如消费偏好与行为不一致）
- 自动修正不一致的响应
- 提供一致性评分

技术路径: 
- 内部一致性检测（基于Persona的态度推断）
- 迭代校准机制（Round 1生成 → Round 2检查 → Round 3修正）
```

### 能力4: 可靠性评估能力（质量保证）
```
定义: 系统能自动评估输出质量，识别不可靠响应

具体要求:
- 内部一致性评分
- 外部分布对齐评估
- 自动触发人工审核

技术路径: 四维评估框架
- 分布相似度（LLM-S³）
- 分析有效性
- 行为一致性
- 交互真实性
```

## 三、核心洞察

> **"虚拟用户 = 统计分布 × 个体知识 × 一致性机制"**

三个要素缺一不可：
- **无统计分布**：Persona是"平均人"，无法体现群体差异
- **无个体知识**：Persona是"空壳"，回答泛泛而谈
- **无一致性机制**：Persona是"随机人"，问答之间矛盾

**协同效应**：三者相互增强
- 统计分布提供群体基准
- 个体知识提供差异化深度
- 一致性机制保证可靠性

## 四、问题定义

### 4.1 研究问题

**核心问题**: 如何构建能够可靠模拟真实人群调研响应的虚拟用户系统？

**子问题**:
1. 如何让LLM感知并学习人口学群体的态度分布？
2. 如何为Persona注入领域知识以增强回答深度？
3. 如何保证虚拟用户在多问题间的一致性？
4. 如何自动评估虚拟用户响应的可靠性？

### 4.2 研究目标

**方法论目标**:
- 提出首个"统计×知识×一致性"三元框架
- 设计四模块系统实现该框架
- 四维评估验证框架有效性

**实证目标**:
- 在4领域21数据集上验证
- 与12个基线对比，显著超越
- 消融实验验证各模块贡献
- 泛化实验验证框架通用性

**应用目标**:
- 相比传统调研，节约70%成本
- 相比传统调研，缩短90%周期
- 响应质量达到可接受标准（分布相似度 > 0.8）
