# P1: 问题定义 - 跨问题态度一致性

## 研究背景

### 传统调研的局限性

调查研究长期以来一直是社会科学、市场研究和政策分析的基石。然而，传统调查方法面临以下挑战：
- **高成本**：招募和补偿真实参与者需要大量资金
- **时间周期长**：从设计到数据收集往往需要数周甚至数月
- **可扩展性有限**：难以快速测试大量假设或进行大规模参数扫描

### LLM虚拟用户的兴起

近年来，大型语言模型（LLMs）为虚拟用户模拟开辟了新可能性：
- **成本效益**：合成数据生成成本远低于真实数据收集
- **快速迭代**：可以在几小时内完成多次实验
- **可控性**：可以精确控制人口统计学特征和实验条件

然而，现有方法存在一个根本性缺陷...

## 核心问题识别

### 问题陈述："分布匹配 ≠ 行为真实"

**当前方法的局限**：
- 现有LLM虚拟用户方法主要优化**分布匹配**（distribution matching）
- 确保虚拟用户的聚合响应模式与真实人群相匹配
- 但忽略了**个体层面的行为一致性**

**关键洞察**：
> 可靠的虚拟用户不仅需要匹配总体分布，还必须在相关问题之间保持一致的逻辑和态度。

### 具体矛盾示例

**示例1：政治态度不一致**
```
问题1：您的党派认同是什么？
回答：民主党

问题2：您对当前经济状况的评价？
回答：很好/改善

问题3：您在总统选举中投票给谁？
回答：特朗普（共和党候选人）
```
问题：民主党认同 + 经济好评 → 应该支持民主党候选人，但回答却是共和党候选人。

**示例2：社会经济态度不一致**
```
问题1：您的最高教育程度？
回答：研究生

问题2：您的家庭年收入？
回答：低于3万美元
```
问题：研究生教育通常与高收入相关，这种组合在真实人群中较为罕见。

## 研究问题

### RQ1: 现有方法的跨问题一致性评估
现有LLM虚拟用户方法在跨问题态度一致性方面表现如何？它们是否存在系统性的一致性问题？

### RQ2: 一致性基准的构建
如何构建一个系统性评估跨问题一致性的基准（benchmark）？需要包含哪些类型的约束？

### RQ3: 一致性度量指标的设计
如何量化虚拟用户响应的跨问题态度一致性？如何设计一个既有效又可解释的度量指标？

### RQ4: 一致性约束方法的开发
如何开发一种在生成分布匹配响应的同时，确保跨问题一致性的方法？

### RQ5: 分布匹配与一致性的权衡
如何在分布匹配和一致性之间取得最佳平衡？是否存在固有的权衡关系？

## 核心概念定义

### 跨问题一致性（Cross-Question Consistency）
虚拟用户在回答相关问题（逻辑上或语义上相关）时，其响应之间保持逻辑连贯和态度一致的程度。

### 分布匹配（Distribution Matching）
虚拟用户响应的边际分布与真实人群响应分布的匹配程度，通常用KL散度、JS距离等度量。

### 行为真实性（Behavioral Authenticity）
虚拟用户响应在个体层面表现出与真实人类相似的逻辑连贯性、态度稳定性和行为模式。

### 一致性约束（Consistency Constraint）
定义两个问题之间期望一致性关系的规则，包括逻辑约束、语义约束和价值观约束。

## 研究意义

### 理论意义
1. **拓展虚拟用户评估维度**：从单一的分布匹配拓展到多维度评估框架
2. **建立一致性理论框架**：为LLM模拟人类行为的一致性要求提供理论基础
3. **连接心理学与AI**：借鉴心理学中的态度一致性理论，应用于LLM评估

### 实践意义
1. **提高虚拟用户可靠性**：使LLM虚拟用户更适合高风险决策支持
2. **降低调研成本**：减少对真实参与者的依赖，同时保持数据质量
3. **加速政策模拟**：支持更快速、更可靠的政策影响评估
4. **改善市场研究**：提供更可信的消费者行为模拟

## 研究范围与边界

### 研究范围
- **数据集**：ANES 2020, GSS 2018（美国政治/社会调查数据）
- **问题类型**：单选题、态度量表题
- **一致性类型**：逻辑一致性、语义一致性、价值观一致性
- **评估方法**：自动评估 + 人工评估

### 研究边界
- **不包含**：开放式问题、多轮对话、纵向时间序列
- **限定**：英语调查数据、美国政治社会背景
- **假设**：一致性约束可以从真实数据中识别和验证

## 预期贡献

### 1. CQCB基准（Cross-Question Consistency Benchmark）
- 首个专门针对虚拟用户跨问题一致性的评估基准
- 包含50+预定义一致性约束，涵盖政治、社会、人口统计领域
- 基于ANES和GSS等权威调查数据

### 2. ACS指标（Attitude Consistency Score）
- 量化跨问题态度一致性的标准化指标
- 支持领域级细粒度评估
- 提供统计显著性检验框架

### 3. ConsistAgent方法
- 基于记忆的一致性约束解码方法
- 在保持分布匹配的同时提高一致性
- 可配置的约束强度参数支持灵活性

### 4. 核心洞察验证
- 实证证明"分布匹配 ≠ 行为真实性"
- 揭示现有方法的系统性局限
- 为虚拟用户研究提供新的评估范式

## 成功标准

### 定量标准
- [ ] ConsistAgent在ACS指标上显著优于基线方法（提升>10%）
- [ ] ConsistAgent保持与基线相当的分布匹配性能（差距<5%）
- [ ] 消融实验验证各模块贡献
- [ ] 人工评估验证响应可信度提升

### 定性标准
- [ ] 论文清晰阐述核心洞察和gap
- [ ] 方法设计可复现、可扩展
- [ ] 实验结果支持主要结论
- [ ] 讨论部分诚实面对局限性和未来工作

## 风险与缓解

| 风险 | 可能性 | 影响 | 缓解措施 |
|------|--------|------|----------|
| 一致性提升导致分布匹配下降 | 中 | 高 | 调整约束强度参数，寻找最优平衡点 |
| 约束定义不够全面 | 高 | 中 | 通过专家验证和文献调研扩展约束集 |
| 人工评估成本过高 | 中 | 中 | 使用小规模但高质量的专家评估 |
| 实验结果不符合预期 | 低 | 高 | 准备多种参数配置和分析角度 |
