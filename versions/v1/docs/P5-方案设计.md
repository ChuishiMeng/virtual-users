# P5: 虚拟用户调研系统方案设计

## 研究主题
**用大模型构造虚拟用户替代传统人工问卷调研**

---

## 1. 研究目标

### 1.1 总体目标
构建基于大语言模型的虚拟用户系统，实现自动化、低成本、高效率的问卷调研替代方案，解决传统问卷调研中的成本高、周期长、样本偏差等问题。

### 1.2 具体目标

#### 短期目标
- **技术验证**：证明 LLM 能够模拟真实用户的行为模式和决策过程
- **原型系统**：构建端到端的虚拟用户调研系统原型
- **准确性基准**：在 3-5 个真实场景中验证系统输出与真实用户调研结果的一致性

#### 长期目标
- **行业应用**：在市场调研、用户体验测试、政策研究等领域推广应用
- **标准制定**：建立虚拟用户调研的质量标准和评估体系
- **人机协同**：探索虚拟用户与真实用户混合调研的最佳实践

### 1.3 成功标准
| 指标 | 目标值 | 说明 |
|------|--------|------|
| 响应相关性 | > 85% | 虚拟用户回答与真实用户回答的语义相关性 |
| 行为一致性 | > 80% | 决策偏好与真实用户群体的一致性 |
| 成本降低 | > 70% | 相比传统问卷调研的成本节约 |
| 时间效率 | < 10% | 完成调研所需时间缩短至传统方法的 10% 以内 |

### 2.2 四步核心流程

#### Step 1: Persona 生成（Persona Generation）
**目标**：构建真实、多元、可解释的虚拟用户画像

**输入**：
- 目标人群描述
- 人口统计数据
- 行为特征标签

**输出**：
- 结构化 Persona 文档
- 包含：背景、偏好、行为模式、决策因素

#### Step 2: 检索增强（Retrieval-Augmented）
**目标**：为 Persona 注入真实世界知识和上下文

**输入**：
- Persona 文档
- 调研问卷/问题
- 知识库（行业报告、用户评论、市场数据）

**输出**：
- 增强的 Persona + 上下文知识
- 相关领域知识检索结果

#### Step 3: 响应生成（Response Generation）
**目标**：基于 Persona 生成符合角色特征的调研响应

**输入**：
- 增强的 Persona
- 具体调研问题
- 上下文信息

**输出**：
- 结构化回答
- 选择题选项
- 开放式反馈

#### Step 4: 可靠性评估（Reliability Assessment）
**目标**：评估响应质量和角色一致性

**输入**：
- 生成的响应
- Persona 定义
- 参考标准（可选）

**输出**：
- 一致性得分
- 置信度指标
- 异常检测报告

### 3.3 响应生成模块

#### 3.3.1 设计理念
结合 PersonaCite 的角色扮演能力和 AlignSurvey 的对齐技术，生成**真实、可信、多样化**的调研响应。

#### 3.3.2 生成策略

**A. 问题类型适配**

| 问题类型 | 生成策略 | 示例 |
|----------|----------|------|
| **单选题** | 基于 Persona 偏好的概率分布采样 | 选择最符合价值观的选项 |
| **多选题** | 阈值筛选 + 相关性约束 | 选择 2-3 个相关选项 |
| **量表题** | 高斯分布 + Persona 偏置 | 偏向正面/负面评价 |
| **开放题** | 个性化生成 + 风格控制 | 简洁/详细，理性/感性 |
| **排序题** | 基于决策模型的偏好排序 | 优先考虑性价比/品质等 |

**B. 多样性控制**
```python
def generate_diverse_responses(persona, question, n=5):
    """
    生成多样化的响应集合
    """
    responses = []
    for i in range(n):
        # 调整温度参数
        temperature = 0.7 + (i * 0.1)
        # 注入随机噪声
        noise = sample_noise(scale=0.1)
        # 生成响应
        response = llm_generate(
            prompt=build_prompt(persona, question, noise),
            temperature=temperature
        )
        responses.append(response)
    return diversity_filter(responses)
```

**C. 不确定性与诚实性**
- 允许 "不知道" / "不确定" 回答
- 对于超出 Persona 知识范围的问题，生成合理的拒绝或不完整回答
- 避免过度自信或完美主义回答

#### 3.3.3 响应质量控制

**质量维度**
1. **相关性**：回答是否切题
2. **一致性**：是否与 Persona 特征一致
3. **真实性**：是否反映真实用户可能的想法
4. **多样性**：不同 Persona 的回答是否有差异
5. **完整性**：是否充分回答了问题

**自动化检查**
```python
class ResponseValidator:
    def validate(self, response, persona, question):
        scores = {
            "relevance": self.check_relevance(response, question),
            "consistency": self.check_consistency(response, persona),
            "realism": self.check_realism(response),
            "diversity": self.check_diversity(response),
            "completeness": self.check_completeness(response, question)
        }
        return scores
```

### 3.4 可靠性评估模块

#### 3.4.1 设计理念
建立**多维度、多层次**的可靠性评估体系，参考 AlignSurvey 的评估框架。

#### 3.4.2 评估维度

**A. 内部一致性评估**
- **Persona-Response 一致性**：回答是否符合 Persona 定义
- **跨问题一致性**：相关问题回答是否矛盾
- **时间稳定性**：重复提问是否得到相似回答

**B. 外部有效性评估**
- **与真实数据对比**：与历史真实调研数据对比
- **专家评估**：领域专家判断响应合理性
- **A/B 测试**：与传统方法结果对比

**C. 系统鲁棒性评估**
- **对抗测试**：恶意输入是否能引发异常
- **边界测试**：极端 Persona 是否产生合理响应
- **分布偏移**：新领域适应性

#### 3.4.3 评估指标体系

| 层级 | 指标 | 计算方法 | 目标值 |
|------|------|----------|--------|
| **个人级** | 角色一致性得分 | Persona-Response 匹配度 | > 0.8 |
| **群体级** | 分布相似度 | KL 散度 / Wasserstein 距离 | < 0.2 |
| **统计级** | 聚合准确度 | 均值/中位数/分位数误差 | < 10% |
| **语义级** | 主题一致性 | NMI / ARI 聚类指标 | > 0.7 |

### 4.2 评估指标详细定义

#### 4.2.1 定量指标

**A. 分布层面**
```python
# 分布相似度
def distribution_similarity(real_dist, virtual_dist):
    """
    计算真实分布与虚拟分布的相似度
    """
    # KL 散度
    kl = entropy(real_dist, virtual_dist)
    # Wasserstein 距离
    wasserstein = wasserstein_distance(real_dist, virtual_dist)
    # JS 散度
    js = jensenshannon(real_dist, virtual_dist)
    return {"kl_divergence": kl, "wasserstein": wasserstein, "js_divergence": js}
```

**B. 聚合层面**
```python
# 聚合指标误差
def aggregation_error(real_responses, virtual_responses):
    """
    计算聚合统计量的误差
    """
    metrics = {}
    # 均值误差
    metrics["mean_error"] = abs(np.mean(real_responses) - np.mean(virtual_responses))
    # 中位数误差
    metrics["median_error"] = abs(np.median(real_responses) - np.median(virtual_responses))
    # 分位数误差
    for q in [0.25, 0.75]:
        metrics[f"q{q}_error"] = abs(np.quantile(real_responses, q) - np.quantile(virtual_responses, q))
    return metrics
```

**C. 个体层面**
```python
# Persona 一致性得分
def persona_consistency_score(persona, responses):
    """
    评估响应与 Persona 的一致性
    """
    # 提取 Persona 关键特征
    persona_traits = extract_persona_traits(persona)
    # 分析响应中的特征
    response_traits = extract_response_traits(responses)
    # 计算匹配度
    score = cosine_similarity(persona_traits, response_traits)
    return score
```

## 5. 创新点

### 5.1 技术创新

#### 创新点 1：多阶段 Persona 生成框架
- **问题**：现有方法直接生成 Persona，缺乏验证和迭代
- **创新**：
  - 采用"生成→验证→修正"的迭代流程
  - 引入 Persona 一致性检查器
  - 支持 Persona 演化和更新
- **价值**：提升 Persona 真实性和可靠性

#### 创新点 2：动态检索增强策略
- **问题**：静态知识库无法适应不同调研场景
- **创新**：
  - 基于 Persona 和问题的动态知识检索
  - 多源知识融合与去重
  - 知识时效性管理
- **价值**：提升响应的准确性和时效性

#### 创新点 3：置信度感知的响应生成
- **问题**：LLM 倾向于生成过度自信的回答
- **创新**：
  - 引入不确定性估计
  - 支持"不知道"/"不确定"回答
  - 基于置信度的响应筛选
- **价值**：提升响应真实性和可信度

### 5.2 应用创新

#### 创新点 4：端到端调研自动化
- **问题**：传统调研流程割裂，效率低下
- **创新**：
  - 从问卷设计到报告生成的全流程自动化
  - 支持 A/B 测试、多轮调研
  - 实时结果分析和可视化
- **价值**：大幅降低调研成本和时间

#### 创新点 5：人机混合调研模式
- **问题**：完全替代真实用户存在风险
- **创新**：
  - 虚拟用户快速探索 → 真实用户验证
  - 自适应采样：基于虚拟结果优化真实样本
  - 置信度分级：高置信问题直接用虚拟结果
- **价值**：平衡效率与可靠性

### 5.3 理论创新

#### 创新点 6：虚拟用户可靠性理论框架
- **问题**：缺乏评估虚拟用户有效性的理论框架
- **创新**：
  - 定义"虚拟效度"（Virtual Validity）概念
  - 建立多层次评估指标体系
  - 提出可靠性边界条件
- **价值**：为领域建立理论基准

#### 创新点 7：Persona 可解释性方法
- **问题**：虚拟用户决策过程不透明
- **创新**：
  - 生成决策理由（Rationale）
  - 可视化 Persona 激活路径
  - 支持事后解释和审计
- **价值**：提升系统可信度和可接受度
