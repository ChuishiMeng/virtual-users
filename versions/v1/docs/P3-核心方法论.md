### 📄 Paper 1: User Behavior Simulation with LLM-based Agents

**arXiv**: 2306.02552 | **时间**: 2024年2月 | **领域**: cs.IR, cs.AI

#### 核心方法

```
┌─────────────────┐
│  LLM-based      │
│  Agent Framework│
├─────────────────┤
│ • Profile Module│ → 用户画像（背景、偏好）
│ • Memory Module │ → 历史交互记忆
│ • Action Module │ → 行为决策
└─────────────────┘
         ↓
┌─────────────────┐
│  Sandbox        │
│  Environment    │
├─────────────────┤
│ • 模拟真实环境  │
│ • 多Agent交互   │
│ • 行为记录      │
└─────────────────┘
```

#### 数据集/评估
- 沙盒环境测试
- 与真实用户行为对比

#### 主要发现
- 模拟行为与真实人类非常接近
- 可研究社会现象（信息茧房、从众效应）

#### 局限性
- ⚠️ 未解决跨人群/方言的泛化问题
- 无来源约束

### 📄 Paper 2: Lost in Simulation

**arXiv**: 2601.17087 | **时间**: 2026年1月 | **领域**: cs.HC, cs.AI

#### ⚠️ 核心发现：LLM模拟用户不可靠

| 问题 | 具体表现 |
|------|----------|
| **缺乏鲁棒性** | 不同LLM间成功率差异达 9% |
| **系统性校准错误** | 低估困难任务，高估中等任务 |
| **方言歧视** | AAVE（非裔英语）成功率更低 |
| **年龄复合效应** | 歧视随年龄增长加剧 |
| **失败模式不同** | 与真实用户的失败模式不一致 |

#### 数据集/评估
- **τ-Bench** 零售任务
- **用户研究**：美国、印度、肯尼亚、尼日利亚

#### 启示
**这是需要解决的核心问题！**

### 📄 Paper 3: Generative Agents (Stanford)

**arXiv**: 2304.03442 | **时间**: 2023年4月 | **领域**: UIST 2023 Best Paper

#### 核心架构

```
┌─────────────────────────────────────────┐
│           Generative Agent              │
├─────────────────────────────────────────┤
│  Memory Stream (记忆流)                  │
│  • 存储所有观察和经历                    │
│  • 时间戳索引                           │
├─────────────────────────────────────────┤
│  Reflection (反思)                       │
│  • 周期性总结高层次观察                  │
│  • 形成"关于自己的思考"                 │
├─────────────────────────────────────────┤
│  Planning (规划)                         │
│  • 基于反思和记忆制定行动计划            │
│  • 动态调整                              │
└─────────────────────────────────────────┘
```

#### 数据集/评估
- **Smallville沙盒**：25个AI agent社区
- 人类评估 believability

#### 主要发现
- 可模拟个体行为和群体涌现现象
- 记忆流+反思+规划是关键技术

#### 局限性
- 无来源约束
- 沙盒规模有限

### 📄 Paper 4: SimUSER

**arXiv**: 2504.12722 | **时间**: 2025年4月 | **领域**: cs.IR, cs.AI

#### 核心方法：两阶段框架

```
Phase 1: Self-Consistent Persona Matching
┌─────────────────────────────────────┐
│ 历史数据 → 提取Persona → 增强Profile │
│ （背景、性格、偏好）                │
└─────────────────────────────────────┘

Phase 2: Recommender System Evaluation
┌─────────────────────────────────────┐
│ 用户Agent模块：                      │
│ • Persona Module - 身份              │
│ • Memory Module - 记忆               │
│ • Perception Module - 感知          │
│ • Brain Module - 决策               │
└─────────────────────────────────────┘
```

#### 数据集/评估
- 推荐系统数据集
- 缩略图效果、曝光效应、评论影响测试

#### 主要发现
- 离线A/B测试 → 优化参数 → 真实世界改进

#### 局限性
- 仅针对推荐系统
- 无来源约束

### 📄 Paper 5: PersonaCite

**arXiv**: 2601.22288 | **时间**: 2026年1月 | **领域**: cs.HC, cs.AI

#### 核心方法：检索增强交互

```
传统方法（不可验证）:
Prompt → LLM生成 → 说服性强但无来源

PersonaCite（可验证）:
┌─────────────────────────────────────┐
│ 对话 → 检索VoC数据 → 约束响应      │
│      ↓                              │
│ • 仅基于检索到的证据回答            │
│ • 证据不足时显式拒绝                │
│ • 响应级别来源归因                  │
└─────────────────────────────────────┘
```

#### 数据集/评估
- 14位行业专家半结构化访谈
- 部署研究

#### 主要发现
- 可验证性是关键
- Persona Provenance Cards 文档模式

#### 局限性
- 依赖VoC数据质量

### 📄 Paper 7: LLM-S³ (Large Language Model-based Sociodemographic Survey Simulation) ⭐ 直接相关

**arXiv**: 2509.06337 | **时间**: 2025年9月 | **领域**: cs.AI

```
┌─────────────────────────────────────┐

├─────────────────────────────────────┤

│ 数据集：                              │
│ • 11个真实公共数据集                 │
```

#### 主要发现

### 📄 Paper 8: AlignSurvey ⭐ 评估基准

**arXiv**: 2511.07871 | **时间**: 2025年11月 | **领域**: cs.CL

#### 核心方法

```
┌─────────────────────────────────────────┐
│          AlignSurvey Benchmark          │
├─────────────────────────────────────────┤
│ 四个任务（对应关键调查阶段）：           │
│ 1. Social Role Modeling (社会角色建模)  │
│ 2. Semi-structured Interview Modeling   │
│    (半结构化访谈建模)                   │
│ 3. Attitude Stance Modeling             │
│    (态度立场建模)                       │
│ 4. Survey Response Modeling             │
│    (调查响应建模)                       │
├─────────────────────────────────────────┤
│ 评估指标：                               │
│ • 对齐保真度 (Alignment Fidelity)       │
│ • 一致性 (Consistency)                  │
│ • 公平性 (Fairness)                     │
│ • 个体和群体层面                        │
├─────────────────────────────────────────┤
│ 数据集：                                 │
│ • Social Foundation Corpus              │
│   - 44K+ 访谈对话                       │
│   - 400K+ 结构化调查记录                │
│ • AlignSurvey-Expert (ASE) 专家标注     │
│ • 两项全国代表性调查                    │
├─────────────────────────────────────────┤
│ 模型：SurveyLM系列                       │
│ • 两阶段微调的开源LLM                   │
│ • 领域特定对齐评估参考模型              │
└─────────────────────────────────────────┘
```

#### 主要发现
- **首个完整社会调查流程基准**
- 关注人口多样性
- 提供开源数据集和模型

#### 启示
- **可作为核心评估基准**
- 四阶段任务设计可参考

### 📄 Paper 9: HumanStudy-Bench ⭐ 参与者模拟

**arXiv**: 2602.00685 | **时间**: 2026年2月 | **领域**: cs.AI

#### 核心方法

```
┌─────────────────────────────────────────┐
│        HumanStudy-Bench                 │
├─────────────────────────────────────────┤
│ 核心观点：                              │
│ 参与者模拟 = Agent设计问题              │
│ Agent = 基础模型 + 规格说明             │
│         (base model + specification)    │
├─────────────────────────────────────────┤
│ 执行流程：                              │
│ Filter → Extract → Execute → Evaluate  │
│                                         │
│ 1. Filter: 筛选相关实验                 │
│ 2. Extract: 提取实验协议               │
│ 3. Execute: 重放试验序列               │
│ 4. Evaluate: 运行原始分析流程          │
├─────────────────────────────────────────┤
│ 评估指标：                              │
│ • 人类-Agent行为一致性                  │
│ • 科学推断层面的保真度                  │
├─────────────────────────────────────────┤
│ 数据集：                                │
│ • 12项基础研究                          │
│ • 个体认知、策略交互、社会心理学        │
│ • 6,000+ 试验                           │
│ • 人类样本：数十到2,100+参与者          │
└─────────────────────────────────────────┘
```

#### 主要发现
- Agent设计影响模拟结果
- 需要区分基础模型能力和实验实例化
- 保持原始统计流程端到端

#### 启示
- Agent设计是关键
- 评估指标可参考
