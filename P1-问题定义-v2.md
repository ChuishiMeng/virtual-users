## 一、Background（研究背景）

### 1.1 场景描述

**社会科学研究现状**：
- 传统研究依赖人类参与（问卷、实验）
- 目标：测试理论假设、理解社会现象、预测集体结果

**LLM新机遇**：
- LLM展现人类级推理和规划能力
- 可作为自主Agent模拟人类行为
- "算法保真度"（Algorithmic Fidelity）成为可能

### 1.2 面临瓶颈

| 瓶颈 | 描述 | 具体问题 |
|------|------|---------|
| **成本高昂** | 人类研究耗时耗资 | 招募代表性样本困难 |
| **难以规模化** | 样本量受限 | 统计显著性不足 |
| **伦理风险** | 涉及人类被试 | 隐私、知情同意等问题 |
| **过度一致性** | 现有LLM方法问题 | Persona Prompting 导致"虚假共识" |

## 二、What（研究目标）

### 2.1 Goal（定位与理念）

**核心定位**：
> **构建一致性水平与真人校准的虚拟用户模拟系统**

**关键转向**（vs 原版本）：

| 原定位 | 新定位 | 原因 |
|--------|--------|------|
| "强制一致性" | **"校准一致性"** | 人类本身不一致(60-80%)，强制100%是模拟idealized agent |
| "态度一致性系统" | **"一致性-真实度前沿"** | 需要平衡一致性vs真实性vs多样性 |

**理念**：
- **可控性**：可调节的一致性强度（α参数）
- **真实性**：与真人一致性水平匹配
- **可解释**：清晰的约束规则和评估指标

### 2.2 Results（预期成果）

**量化指标**：

| 指标 | 目标值 | 验证方法 |
|------|--------|---------|
| **Calibration Error** | |ACS_simulated - ACS_human| < 5% | ANES Panel真人数据对比 |
| **Distribution Match** | KL Divergence < 0.1 | 分布相似度检验 |
| **Diversity Index** | Response entropy > human baseline | 避免 mode collapse |
| **Over-consistency Detection** | 识别 ACS > 90% 的"不真实"案例 | Failure case 分析 |

**产出物**：

| 产出 | 描述 | 学术价值 |
|------|------|---------|
| **CQCB基准** | 跨问题一致性约束集（50条） | 公开数据贡献 |
| **Calibration Controller** | 一致性校准算法 | 方法贡献 |
| **ACS-Improved指标** | Diversity-adjusted ACS | 评估贡献 |
| **Human Consistency Study** | 真人一致性baseline数据 | 实证贡献 |

## 三、Why（研究意义与挑战）

### 3.1 理论讨论：一致性 vs 真实性

**核心问题**（审稿人质疑）：
> "如果人类本身就不一致，强制一致性是在模拟什么？"

**理论支撑**：

| 理论 | 关键发现 | 本工作启示 |
|------|---------|-----------|
| **Converse 1964** (Belief Systems) | 大众政治态度不稳定(60-70%一致) | 不要追求100%一致性 |
| **Festinger 1957** (Cognitive Dissonance) | 人类追求一致但无法完全达成 | 一致性是心理驱动，非行为规律 |
| **Simon 1957** (Bounded Rationality) | 人类决策能力有限 | "适度不一致"是人类特性 |

**三层一致性定义**：

| 层级 | 内容 | 人类一致性水平 | 模拟目标 |
|------|------|---------------|---------|
| **Identity层** | 人口学、核心价值观 | 90-95% | 强一致（必须保证） |
| **Preference层** | 产品偏好、态度倾向 | 70-80% | 中一致（与真人校准） |
| **Momentary层** | 情绪、情境响应 | 40-60% | 弱一致（保留噪声） |

**何时不应强制一致性**：
- 探索性行为（尝试新品类）
- 情境漂移（不同场景不同偏好）
- 学习演化（随时间改变态度）

### 3.2 Values（应用价值）

#### 3.2.1 市场调研场景

| 场景 | 过度一致问题 | 校准一致价值 |
|------|-------------|-------------|
| **产品测试预测** | 永远给好评 → 误判市场 | 预测分布匹配真人 → 决策可靠 |
| **广告效果评估** | 虚假共识 → 错误选择 | 保留不确定性 → 置信区间预测 |
| **市场规模预测** | 预测偏差35% | 预测误差降至12% |

#### 3.2.2 推荐系统场景

| 场景 | 过度一致问题 | 校准一致价值 |
|------|-------------|-------------|
| **新用户冷启动** | 推荐多样性差 | 模拟真人探索行为 |
| **A/B测试加速** | 无法模拟"偶然尝试" | 快速生成预测评分矩阵 |

#### 3.2.3 社会科学模拟

| 场景 | 过度一致问题 | 校准一致价值 |
|------|-------------|-------------|
| **选举预测** | 100%党派忠诚 → 极端化预测 | 5-7%跨党派投票 → 真实选举 |
| **舆论涌现** | 无法产生分歧 | 模拟观点分歧、群体极化 |

#### 3.2.4 学术价值

- **填补空白**：首个"校准一致性"框架（而非强制一致性）
- **新评估范式**：ACS + Diversity + Human Calibration 三维评估
- **理论贡献**：Consistency-Fidelity Frontier（一致性-真实度前沿）

### 3.3 Challenges（研究挑战）

| 挑战 | 描述 | 难度 | 为什么难 |
|------|------|------|---------|
| **C1: Human Baseline测量** | 如何量化真人的一致性水平？ | 高 | 需要纵向数据(Panel)，真人本身有噪声 |
| **C2: 一致性校准算法** | 如何让模拟ACS匹配真人ACS？ | 高 | 不同层级需要不同校准强度 |
| **C3: Trade-off理论** | 如何形式化一致性与真实性的权衡？ | 高 | 需要Pareto前沿分析 |
| **C4: 评估指标改进** | 如何避免ACS的mode collapse问题？ | 中 | 需要Diversity-adjusted设计 |
| **C5: 下游任务验证** | 如何证明"适度一致"有应用价值？ | 中 | 需要对比实验 |

## 四、How（研究方法）

### 4.1 总体框架（修订版）

```
┌─────────────────────────────────────────────────────────┐
│           Calibrated Consistency Framework               │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  ┌─────────────┐    ┌─────────────┐    ┌─────────────┐ │
│  │ Human Baseline│ → │ Calibration │ → │ACS-Improved │ │
│  │ (真人一致性) │    │ Controller  │    │ (评估指标)  │ │
│  └─────────────┘    └─────────────┘    └─────────────┘ │
│         │                  │                  │        │
│         ▼                  ▼                  ▼        │
│   ANES Panel数据     分层校准(α_L1/α_L2/α_L3)  Diversity-adjusted │
│   MovieLens评分      Dimension-aware         Human-calibrated  │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 4.2 模块一：Human Consistency Baseline

**功能**：测量真人一致性水平作为校准目标

**数据源**：

| 数据 | 内容 | 一致性测量方法 | 预期值 |
|------|------|---------------|-------|
| **ANES Panel** | 政治态度(2016→2020) | Exact Match rate | ~65% |
| **MovieLens** | 电影评分(多次评分) | Correlation | ~0.75 |
| **ESM日记** | 情绪状态(日内多次) | Exact Match | ~45% |

### 4.3 模块二：Calibration Controller

**功能**：让模拟用户一致性匹配真人水平

**分层校准策略**：

```python
# 三层一致性控制
α_identity = 0.95    # 人口学必须稳定
α_preference = 0.70  # 与真人偏好一致水平匹配
α_momentary = 0.50   # 情绪响应保留噪声

# 校准目标
ACS_target = ACS_human  # 让模拟ACS = 真人ACS（而非追求100%）
```

### 4.4 模块三：ACS-Improved（改进版评估指标）

**原有ACS问题**：
- 无法区分"真一致"和"mode collapse"
- baseline ACS=0.917但实际是重复响应

**改进设计**：

```python
# Dimension-aware ACS（分层）
ACS = {
    ACS_identity: 人口学稳定性
    ACS_preference:态度一致性
    ACS_behavior:行为预测一致性
}

# Diversity-adjusted（避免mode collapse）
ACS_adj = ACS × (1 - entropy_penalty)
其中 entropy_penalty = 
    0  if diversity > human_baseline
    1  if diversity = 0 (mode collapse)

# Human-calibrated（与真人校准）
Calibration_Error = |ACS_simulated - ACS_human|
目标：< 5% 偏差
```

## 五、How-Why（方法论证）

### 5.1 核心Insight

| 挑战 | 解决方案 | 为什么这样解决 |
|------|---------|---------------|
| **C1: Human Baseline** | ANES Panel + MovieLens | 真人一致性是校准目标，不是追求100% |
| **C2: 校准算法** | 分层校准(α_L1/L2/L3) | 不同层级人类一致性不同 |
| **C3: Trade-off理论** | Consistency-Fidelity Frontier | 形式化权衡关系 |
| **C4: ACS改进** | Diversity-adjusted | 避免"虚假一致" |
| **C5: 下游验证** | 对比实验 | 证明应用价值 |

### 5.2 与相关工作对比（修订版）

| 维度 | Argyle 2023 (Silicon Sampling) | Park 2024 (1000 People) | **本工作** |
|------|-------------------------------|------------------------|-----------|
| **核心目标** | 分布匹配 | Persona一致性 | **校准一致性** |
| **一致性机制** | ❌ 无 | ⚠️ 隐式访谈数据 | **显式分层校准** |
| **评估指标** | KL Divergence | Persona fidelity | **ACS + Diversity + Calibration** |
| **Human Baseline** | ❌ 无 | ⚠️ 隐式 | **✅ 显式测量真人ACS** |
| **Trade-off分析** | ❌ 无 | ❌ 无 | **✅ Pareto前沿** |

**关键差异化**：
> "我们不追求'完美一致的虚拟用户'，而是追求'一致性水平与真人匹配的虚拟用户'"
