## 📋 基本信息

**研究主题**: 评估AI代理在社会科学实验中的参与者模拟能力

**核心问题**: 
- LLM代理能否可靠模拟人类实验参与者？
- 代理设计如何影响模拟保真度？
- 如何在科学推断层面评估对齐度？

**核心贡献**:
1. **HumanStudy-Bench基准**: 12个经典人类实验的重建
2. **Filter-Extract-Execute-Evaluate流水线**: 完整实验重建
3. **概率对齐分数（PAS）**: 科学推断层面评估
4. **效应一致性分数（ECS）**: 数据层面评估

**规模**: 6,000+ 试验，参与者从数十到2,100+

---

## 🎯 研究动机

### LLM模拟的问题

**当前应用的挑战**：
1. **不稳定**：响应分布不稳定（Santurkar et al., 2023）
2. **人口异质性不敏感**：缺乏对人口多样性的敏感性（Bisbee et al., 2024）
3. **设计敏感**：过度敏感于实验设计选择（提示词、采样设置）（Loya et al., 2023; Sclar et al., 2024）

**现有评估的局限**：
- 评估原始模型在定制任务上的行为
- 测试固定提示词能否定性复现经典效应
- **混淆**：基础模型能力 vs 实验实例化

### 代理设计的重要性

**核心观点**：参与者模拟是**代理设计问题**

**代理定义**：
```
Agent = Base Model + Agent Specification
```

**Agent Specification包括**：
- 角色和任务框架
- 人口统计属性
- 目标、记忆
- 辅助机制（工具、技能）

**关键洞察**：
- 这些规范不是中立的包装器
- 编码了行为假设
- 即使基础模型固定，也能定性改变结果

## 💡 核心方法

### 1. 完整流水线

```
Filter → Extract → Execute → Evaluate
(筛选)  (提取)    (执行)     (评估)
```

#### Stage 1: Filter（筛选）

**目标**：从文献中筛选人类实验

**筛选标准**：
- 包含明确实验设计
- 有人类参与者数据
- 可重建实验流程

#### Stage 2: Extract（提取）

**提取内容**：
- 研究ID、现象、发现
- 统计检验（t-test, χ², ANOVA等）
- 样本量、均值、标准差
- p值、效应量

#### Stage 3: Execute（执行）

**执行模块**：
- 创建试验（create_trials）
- 构建提示词（build_trial_prompt）
- 代理响应（agent.generate）

#### Stage 4: Evaluate（评估）

**评估指标**：
1. **Probability Alignment Score (PAS)**
2. **Effect Consistency Score (ECS)**

### 2. 12个经典实验

**个体认知类**：
1. Anchoring Effect（锚定效应）
2. Framing Effect（框架效应）
3. Conjunction Fallacy（合取谬误）
4. Representativeness Heuristic（代表性启发式）

**策略交互类**：
5. p-Beauty Contest Game（p-美赛博弈）
6. Prisoner's Dilemma（囚徒困境）
7. Ultimatum and Dictator Games（最后通牒和独裁者博弈）
8. Trust and Reciprocity Game（信任和互惠博弈）

**社会心理学类**：
9. Intentional Action and Side-Effects（意图行动和副作用）
10. Forming Impressions of Personality（印象形成）
11. Social Categorization（社会分类）
12. Pluralistic Ignorance（多元无知）

**总规模**：6,000+ 试验

### 3. 评估指标详解

#### Probability Alignment Score (PAS)

**定义**：测量科学推断层面的对齐

**公式**：
```
PAS = π_human · π_agent + (1 - π_human) · (1 - π_agent)
```

**理论基础**：
- **频率学派视角**：方差缩减（收缩估计器）
- **贝叶斯学派视角**：最小贝叶斯风险

**直觉解释**：
- PAS测量"科学复现率"
- 回答："如果我用这个代理复现假设检验，会得到与人类相同的结果吗？"
- 这是**现象级别**的指标

#### Effect Consistency Score (ECS)

**定义**：测量数据层面的保真度

**评估内容**：
- 效应量的一致性
- 数据分布的结构相似性

**直觉解释**：
- ECS回答："代理的效应量和数据分布是否准确反映人类的？"
- 这是**数据级别**的更严格指标

---

### 4. 代理设计变体

#### A1: Blank（空白）
- 无角色，无人口统计约束

#### A2: Role-Play（角色扮演）
- 明确角色，但无具体人口统计

#### A3: Demographic（人口统计）
- 人口统计约束（年龄、性别、教育、收入）

#### A4: Contextualized Backstory（上下文背景故事）
- 丰富的背景故事，包含经历、态度

## 📊 实验结果

### 1. 总体对齐有限 ⚠️

**关键发现**：**所有模型PAS < 0.6**

**解释**：
- PAS = 0.5 是随机水平
- PAS > 0.6 是合理对齐
- **所有当前LLM < 0.6** → 对齐有限

### 2. 行为极端化 ⚠️⚠️⚠️

**人类行为**：单峰分布（温和）

**LLM行为**：双峰分布（极端）

**示例**（Anchoring Effect）：
- 人类：估算值分布在合理范围
- LLM：要么极低（0），要么极高（最大值），缺少中间值

**影响**：
- 无法复现人类的温和分布
- 效应量失真
- 统计检验结果不同

### 3. 代理设计影响大且非单调

**关键发现**：更复杂的设计 ≠ 更好表现

**非单调效应**：
- 某些研究：A1 > A2 > A3 > A4
- 其他研究：A4 > A2 > A1 > A3
- **无通用最佳设计**

### 4. 领域依赖性强

**不同领域的PAS差异**：

| 领域 | 平均PAS | 说明 |
|------|---------|------|
| **个体认知** | 0.35-0.45 | 相对较好 |
| **策略交互** | 0.25-0.35 | 困难 |
| **社会心理学** | 0.30-0.40 | 中等 |

### 5. 模型规模不保证改进

**关键发现**：大模型不一定更好

**反例**：
- GPT OSS 120b（大）在某些研究上 < GPT 5 Nano（小）
- Claude Haiku（相对小）在多个研究上表现优异

### 6. 多模型集成不可靠

**实验**：混合不同模型（Mixed Models）

**结果**：
- 混合模型PAS ≈ 单模型平均PAS
- 无显著改进
- 有时甚至更差

## 🔍 关键洞察

### 1. 对虚拟用户研究的启示 ⭐⭐⭐⭐⭐

**最严重警告！**

**核心问题**：
1. ✅ **对齐有限**：PAS < 0.6，远低于可接受水平
2. ⚠️ **行为极端化**：双峰 vs 人类单峰
3. ⚠️ **设计敏感性**：提示词、采样设置影响大
4. ⚠️ **领域依赖**：无通用解决方案

**与P11+P12+P14的呼应**：
- P11（人口偏见）+ P12（系统性不可靠）+ P14（顺序偏见）+ P18（对齐有限）

**综合结论**：
- ❌ **不能直接用LLM替代人类受访者**
- ❌ **不能假设LLM行为分布与人类相似**
- ⚠️ **需要真实数据校准和验证**

### 2. 评估方法的价值

**PAS的优势**：
1. ✅ **科学推断层面**：评估是否得到相同科学结论
2. ✅ **考虑不确定性**：考虑人类基线的不确定性
3. ✅ **统一框架**：将不同统计检验映射到共同概率空间
4. ✅ **可解释**：直觉清晰（"科学复现率"）

**与P13 PAS/FAS的对比**：
- P13的PAS/FAS用于问卷评估
- P18的PAS用于实验评估
- 两者提供多层次评估框架

### 3. 代理设计教训

**关键洞察**：
1. ⚠️ **过度约束有害**：复杂背景故事可能引入偏见
2. ⚠️ **领域特定**：不同研究领域需要不同设计
3. ⚠️ **无银弹**：无通用最佳代理设计

**实践建议**：
1. ✅ **从简单开始**：A1 Blank作为基线
2. ✅ **领域适配**：针对具体研究领域优化
3. ✅ **迭代设计**：测试多种设计，选择最佳
4. ✅ **真实数据验证**：用人类数据校准

## 📝 总结

### 优点

1. ✅ **最严格评估**：科学推断层面的PAS
2. ✅ **完整流水线**：Filter-Extract-Execute-Evaluate
3. ✅ **12个经典实验**：跨领域验证
4. ✅ **理论基础扎实**：贝叶斯对齐 + 频率学派验证
5. ✅ **开源代码**：可复现

### 局限

1. ⚠️ **基准范围小**：仅12个研究
2. ⚠️ **模型覆盖少**：仅10个模型
3. ⚠️ **西方背景**：主要是西方实验
4. ⚠️ **实现复杂**：重建实验需要大量工作

### 对本研究的价值

**核心借鉴**（⭐⭐⭐⭐⭐）:
1. **PAS评估**：必须用PAS评估虚拟用户系统
2. **警告信号**：LLM代理对齐有限，需真实数据校准
3. **代理设计**：需测试多种设计，无通用最佳方案
4. **领域适配**：不同研究领域需要不同方法

**必须采取的措施**:
1. ✅ **使用PAS评估**：科学推断层面
2. ⚠️ **真实数据验证**：对照P11+P12+P14+P18警告
3. ✅ **测试多种代理设计**：寻找最佳设计
4. ⚠️ **领域适配**：针对问卷模拟优化
