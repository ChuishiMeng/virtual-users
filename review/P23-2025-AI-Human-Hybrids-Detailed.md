## 💡 核心方法

### 1. 市场研究框架

**LLM应用阶段**：

| 研究阶段 | 探索性（定性）| 描述性（定量）| 因果（A/B测试）|
|---------|-------------|-------------|--------------|
| **研究设计** | 创建讨论指南 | 创建调查问卷 | 创建概念和测试 |
| **样本选择** | 确定样本特征 | 确定样本特征 | 确定样本特征 |
| **数据收集** | 合成受访者 | 合成受访者 | 合成受访者 |
| **数据分析** | 总结/主题提取 | 统计分析+可视化 | 统计分析+可视化 |

### 2. 操作化方法

#### Zero-Shot Prediction

**提示词结构**：
```python
# 系统角色（Persona）
system_role = """
You are a respondent on a survey...
Your characteristics:
- Age: 35
- Gender: Female
- Location: Urban
- Income: $75,000
- ...
"""

# 用户角色（问题）
user_role = """
Question: How likely are you to purchase this product?
Answer format: 1-5 scale
"""

# 助手角色（答案）
response = llm.generate(system_role, user_role)
```

#### Few-Shot Learning

**方法**：在提示词中包含之前的问题-答案对

**示例**：
```python
prompt = """
Previous Q&A:
Q1: How likely to purchase? A: 4
Q2: How much would you pay? A: $15
Q3: How often would you buy? A: Once a month

Current Question: How satisfied are you with the quality?
Answer:
"""
```

**效果**：提升响应一致性和异质性

#### Retrieval-Augmented Generation (RAG)

**方法**：检索相关上下文信息增强生成

**步骤**：
1. 检索：从知识库检索相关信息
2. 增强：将检索信息加入提示词
3. 生成：LLM基于增强提示词生成

**示例**：
```python
# 检索相关访谈记录
relevant_interviews = retrieve_from_knowledge_base(
    query="refrigerated pet food attitudes",
    top_k=5
)

# 增强提示词
prompt = f"""
Context from previous interviews:
{relevant_interviews}

Based on this context, answer the following survey question:
Question: {survey_question}
"""
```

**效果**：显著提升合成数据质量

### 3. 系统架构

**数据生成流程**：
```
1. Persona存储（系统角色）
   ↓
2. 问题存储（用户角色）
   ↓
3. API请求 → OpenAI GPT-4
   ↓
4. 响应处理 + 验证
   ↓
5. 聚合 + 格式化
   ↓
6. 输出合成数据
```

## 🧪 实验1：定性研究（Friendsgiving）

- 相同访谈指南
- 生成合成访谈记录

### 2. 核心发现

#### 数据生成质量

**LLM vs 人类对比**：

| 指标 | LLM生成 | 人类 | 结果 |
|------|---------|------|------|
| **深度** | **更高** | 中等 | ✅ LLM优势 |
| **洞察力** | **更高** | 中等 | ✅ LLM优势 |
| **连贯性** | 相当 | 相当 | = 平局 |
| **信息丰富度** | 相当 | 相当 | = 平局 |

**文本质量指标**（NLP）：
- 可读性：相当
- 词汇多样性：LLM更高
- 句子复杂性：相当

**人类评估**（众包工人）：
- 深度评分：LLM **4.2/5** vs 人类 **3.5/5**
- 洞察力：LLM **4.0/5** vs 人类 **3.3/5**
- 真实感：人类 **4.5/5** vs LLM **3.8/5**

**关键洞察** ⭐⭐⭐：
- ✅ LLM生成**更深入、更有洞察力**
- ⚠️ 人类生成**更真实**

**LLM作为分析师**：

**关键发现** ⭐⭐⭐：

| 方面 | 人类优势 | LLM优势 |
|------|---------|---------|
| **洞察** | 实践经验、文化背景 | 大规模知识、跨领域连接 |

## 🧪 实验2：定量研究（Pet Food）

### 1. 研究设计

**原始研究**（2019年）：
- 主题：冷藏狗粮概念测试
- 受访者：**605人**
- 内容：概念偏好、评估、态度、购买频率

**测试方面**：
- 包装：可重新密封袋 vs 可切片管
- 成分：人类级 vs 新鲜

**LLM复制**：
- GPT-4生成605个合成受访者
- 匹配人口统计特征
- 相同调查问题

### 2. Zero-Shot结果

#### 概念评估

**购买可能性、喜好度、独特性**：

| 指标 | 实际数据 | 合成数据（Zero-Shot）| 差异 |
|------|---------|-------------------|------|
| **购买可能性**均值 | 3.2 | **2.8** | ✅ 方向正确 |
| **购买可能性**标准差 | 1.1 | **0.8** | ⚠️ 方差更小 |
| **喜好度**均值 | 3.5 | **3.5** | ✅ 一致 |
| **喜好度**标准差 | 1.0 | **0.7** | ⚠️ 方差更小 |
| **独特性**均值 | 3.8 | **4.1** | ✅ 方向接近 |
| **独特性**标准差 | 1.2 | **0.9** | ⚠️ 方差更小 |

**关键发现**：
- ✅ **均值方向正确**（正确捕捉总体趋势）
- ⚠️ **方差更小**（响应异质性不足）

#### 态度测量

**7个态度指标**（5分制）：

| 态度 | 实际均值 | 合成均值 | 一致性 |
|------|---------|---------|--------|
| 比其他高端狗粮更多好处 | 3.6 | 3.9 | ⚠️ 差异 |
| 会用作狗的主要食物来源 | 3.4 | **3.4** | ✅ 一致 |
| 方便的新鲜食物服务方式 | 3.8 | 4.1 | ⚠️ 差异 |
| 用最健康的成分制成 | 3.5 | 3.8 | ⚠️ 差异 |
| 最好的质量产品 | 3.7 | 4.0 | ⚠️ 差异 |
| 最安全的狗粮 | 3.6 | 3.9 | ⚠️ 差异 |
| 喂食让我感觉良好 | 3.3 | **3.3** | ✅ 一致 |

**分布特征**：
- 实际数据：分散在1-5
- 合成数据：**集中在4-5**（方差小）

**购买频率**：
- ✅ 某些区间匹配良好（每月2-3次，每月1次）
- ⚠️ 某些区间偏差（每周多次过高，每周1次过低）
- χ²检验：分布**不同**（p < 0.05）

#### Zero-Shot总结

**核心问题**：
1. ⚠️ **响应异质性不足**（标准差小）
2. ⚠️ **分布集中**（集中在高分）
3. ⚠️ **变量间相关性弱**（可靠性问题）

**原因分析**：
- Persona主要基于人口统计
- **人口统计解释偏好能力有限**（众所周知）
- 缺少上下文和历史信息

### 3. Few-Shot Learning改进

**方法**：在提示词中包含之前的问题-答案对

**效果**：

| 指标 | Zero-Shot | Few-Shot | 改进 |
|------|----------|---------|------|
| **异质性**（标准差）| 0.7-0.9 | **0.9-1.1** | ✅ +20-30% |
| **均值一致性** | 2/7一致 | **4/7一致** | ✅ +100% |
| **分布匹配** | 差 | **中** | ✅ 改善 |
| **相关性恢复** | 弱 | **中等** | ✅ 改善 |

**关键改进** ⭐⭐⭐：
- ✅ **异质性提升20-30%**
- ✅ **均值一致性翻倍**
- ✅ **分布更接近真实**

### 4. RAG改进

**方法**：检索增强生成

**检索来源**：
- 之前定性访谈记录
- 公司相关文档
- 市场研究报告

**效果**：

| 指标 | Zero-Shot | Few-Shot | RAG | 最佳 |
|------|----------|---------|-----|------|
| **异质性** | 0.7-0.9 | 0.9-1.1 | **1.0-1.2** | ✅ RAG |
| **均值一致性** | 2/7 | 4/7 | **5/7** | ✅ RAG |
| **相关性** | 弱 | 中等 | **强** | ✅ RAG |
| **分布匹配** | 差 | 中 | **良** | ✅ RAG |

**关键改进** ⭐⭐⭐⭐：
- ✅ **RAG效果最佳**
- ✅ **异质性恢复到接近真实**
- ✅ **均值一致性71%**
- ✅ **变量间相关性恢复良好**

### 2. 技术要点

#### Few-Shot Learning实现

```python
def generate_response_few_shot(persona, question, history, k=3):
    """Few-shot学习生成响应"""
    # 选择最近的k个Q&A对
    recent_qa = history[-k:]
    
    # 构建提示词
    prompt = f"""
Persona: {persona}

Previous Q&A:
{format_qa_pairs(recent_qa)}

Current Question: {question}
Answer:
"""
    
    return llm.generate(prompt)
```

#### RAG实现

```python
def generate_response_rag(persona, question, knowledge_base):
    """RAG生成响应"""
    # 1. 检索相关信息
    relevant_docs = retrieve_relevant(
        query=question,
        knowledge_base=knowledge_base,
        top_k=5
    )
    
    # 2. 增强提示词
    prompt = f"""
Persona: {persona}

Relevant Context:
{format_documents(relevant_docs)}

Question: {question}
Answer:
"""
    
    return llm.generate(prompt)
```
