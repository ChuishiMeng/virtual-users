## 三、一致性保证机制细化

### 3.1 CQCB 约束验证协议

当前 CQCB 有 50 条约束，但缺少正式的验证协议确保约束质量。

#### 验证流程

```
Step 1: 内部一致性检查
├── 逻辑约束: 在 ANES 数据上满足率 ≥ 0.90
├── 语义约束: 在 ANES 数据上满足率 ≥ 0.70
└── 价值约束: 在 ANES 数据上满足率 ≥ 0.75

Step 2: 交叉验证
├── GSS 数据复现检查 (满足率偏差 < 10%)
└── 多标注者一致性 (ICC ≥ 0.80)

Step 3: 约束冲突检测
├── 检查约束对之间是否存在矛盾
└── 构建约束依赖图，检测循环依赖

Step 4: 覆盖度评估
├── 问题覆盖率: 25 个问题中至少 20 个被约束覆盖
├── 约束密度: 平均每个问题被 4+ 条约束关联
└── 领域均衡: 政治/社会/人口三个领域各有 ≥ 12 条约束
```

#### 约束质量标准

| 维度 | 标准 | 验证方法 |
|------|------|---------|
| **有效性** | 真实数据满足率 ≥ 0.80 | ANES/GSS 计算 |
| **不可平凡性** | 随机响应满足率 ≤ 0.30 | 蒙特卡洛模拟 |
| **鉴别力** | 约束能区分好/差模型 | ΔACS ≥ 0.10 |
| **无冲突** | 约束对之间无矛盾 | 图搜索算法 |

#### 关键补充：约束冲突检测

```python
def detect_constraint_conflicts(constraints):
    """
    检测约束之间的潜在冲突
    
    例如：
    - C1: Democrat → Biden
    - C2: Trump supporter → economy bad  
    - C3: Biden supporter → economy good
    这里 C1+C3 可能与某些 persona 冲突
    
    方法：构建约束图，检测矛盾路径
    """
    conflict_pairs = []
    
    for i, c1 in enumerate(constraints):
        for j, c2 in enumerate(constraints):
            if i >= j:
                continue
            
            # 检查是否关联相同问题
            shared_qs = set(c1.questions) & set(c2.questions)
            if not shared_qs:
                continue
            
            # 在真实数据上检查联合满足率
            joint_rate = compute_joint_satisfaction(c1, c2)
            
            if joint_rate < 0.50:
                conflict_pairs.append({
                    'constraint_pair': (c1.id, c2.id),
                    'joint_satisfaction': joint_rate,
                    'severity': 'high' if joint_rate < 0.30 else 'medium'
                })
    
    return conflict_pairs
```

### 3.2 ConsistAgent 算法严谨性补充

#### 问题：当前算法在极端情况下的行为

| 场景 | 当前行为 | 潜在问题 | 改进方案 |
|------|---------|---------|---------|
| 所有权重归零 | 采样随机选项 | 硬约束过度限制 | 退化为软约束 + 警告 |
| α=1.0 + 冲突约束 | 死锁 | 无法生成响应 | 约束优先级排序 |
| 首个问题无记忆 | 均匀采样 | 无约束引导 | 使用群体分布作为先验 |
| 记忆窗口溢出 | LRU 淘汰 | 可能丢失关键约束 | 语义重要性加权淘汰 |

#### 改进算法：带退化的约束解码

```python
def robust_constrained_decode(options, constraints, memory, alpha):
    """
    健壮的约束加权解码
    
    改进:
    1. 权重归零时退化为软约束
    2. 冲突约束按优先级排序
    3. 无记忆时使用群体先验
    """
    weights = {opt: 1.0 for opt in options}
    applied_count = 0
    degraded_count = 0
    
    # 按优先级排序约束
    sorted_constraints = sorted(constraints, key=lambda c: c.priority)
    
    for constraint in sorted_constraints:
        other_q = constraint.get_other_question(current_q_id)
        if other_q not in memory.responses:
            continue
        
        other_response = memory.responses[other_q]
        
        for option in options:
            is_consistent, score = check_consistency(option, other_response, constraint)
            
            if constraint.type == "logical":
                if not is_consistent:
                    # 尝试硬约束
                    weights[option] = 0.0
                    applied_count += 1
            elif constraint.type in ["semantic", "value-based"]:
                adjustment = 1 - alpha * constraint.strength * (1 - score)
                weights[option] *= adjustment
                applied_count += 1
    
    # 退化检查：如果所有选项权重归零
    if sum(weights.values()) == 0:
        degraded_count += 1
        # 退化为：仅使用最高优先级约束
        for constraint in sorted_constraints[:1]:
            other_response = memory.responses.get(constraint.get_other_question(current_q_id))
            if other_response:
                for option in options:
                    _, score = check_consistency(option, other_response, constraint)
                    weights[option] = max(score, 0.01)
    
    # 归一化
    total = sum(weights.values())
    if total > 0:
        weights = {k: v/total for k, v in weights.items()}
    
    return weights, {'applied': applied_count, 'degraded': degraded_count}
```

### 3.3 语义记忆管理（替代简单 LRU）

```python
class SemanticMemoryManager:
    """
    语义感知的记忆管理器
    
    替代简单 LRU，按语义重要性加权
    """
    def __init__(self, max_size=10):
        self.max_size = max_size
        self.responses = {}  # question_id → response
        self.importance = {}  # question_id → importance_score
    
    def update(self, question_id, response, related_constraints):
        """
        更新记忆，考虑语义重要性
        
        importance = f(约束关联数, 约束类型, 时间衰减)
        """
        # 计算重要性
        n_constraints = len(related_constraints)
        has_logical = any(c.type == "logical" for c in related_constraints)
        
        importance = n_constraints * (2.0 if has_logical else 1.0)
        importance += self._recency_bonus()
        
        self.responses[question_id] = response
        self.importance[question_id] = importance
        
        # 如果超出容量，淘汰最低重要性的
        if len(self.responses) > self.max_size:
            min_id = min(self.importance, key=self.importance.get)
            del self.responses[min_id]
            del self.importance[min_id]
    
    def _recency_bonus(self):
        return 1.0  # 最近响应的额外加分
    
    def get_relevant_responses(self, target_question_id, constraints):
        """
        获取与目标问题相关的历史响应
        """
        relevant = {}
        for constraint in constraints:
            for q_id in constraint.questions:
                if q_id != target_question_id and q_id in self.responses:
                    relevant[q_id] = self.responses[q_id]
        return relevant
```

## 四、实验验证方案补充

### 4.1 统计检验方案

P4 已定义了 5 次随机种子重复，但缺少具体的统计检验方案。

#### 检验方法

| 对比 | 检验方法 | 原因 |
|------|---------|------|
| ConsistAgent vs 单个基线 | Wilcoxon signed-rank test | 重复测量非正态 |
| 多方法全局比较 | Friedman test + Nemenyi post-hoc | 多组非参数 |
| 消融实验 | 配对 t 检验 (正态验证后) | 同方法不同配置 |
| 相关性分析 | Spearman rank correlation | 非线性关系 |

#### 多重比较校正

使用 Bonferroni 校正：
- 主实验: 7 方法 × 6 指标 = 42 次比较 → α_corrected = 0.05/42 ≈ 0.0012
- 建议使用 Benjamini-Hochberg FDR 校正（更宽松）

#### 效应量报告

所有显著性检验必须报告效应量：

| 效应量 | 公式 | 解释 |
|--------|------|------|
| Cohen's d | $(\bar{x}_1 - \bar{x}_2) / s_p$ | 组间差异 |
| Rank-biserial correlation | $r = Z / \sqrt{N}$ | Wilcoxon 配套 |
| η² | $SS_{between} / SS_{total}$ | 方差解释率 |

**目标**: Cohen's d > 0.8 或 η² > 0.14 (大效应)
